去写了写机器学习报告。写报告的时候简单研究了一下 word 神秘科技

装了一个新的 Math 字体，这样用 word 写出来的公式就变成和 Latex 一样美观了😍

然后自研了一个 lbn 报告法，就是说插入代码段的时候用这样的样式，给代码加上一个框

这一招是看 lbn 报告学来的。逆向工程之后这里用的是 1x1 表格实现，他能自动断开分页

要求是写 5 个实验，后天之前交，但是晚上只写了一个实验

不想写报告了，遂来写日寄，结果日寄刚写到一半（上面引用那里），冰岩的集群爆了，于是赶紧过去修集群

2c2g 的最小的机器卡住了，导致整个集群 api 不可用，都 504 了。简单重启一下机器其实就能解决

但是这个问题已经出现了若干次，之前爆炸了之后给这个结点加上了污点。这次为了更加一劳永逸的解决时不时就要重启机器的问题，我打算把这个小机器踢出集群得了

ssh 上了这个机器，很快啊，一下就把 k3s 给卸了

然后整个集群再次爆炸

我原本计划的是先给他卸了，然后重装 k3s 变成一个 worker。但是在我卸完 k3s，集群 api 不可用之后，另一个人去剩下的一个 master 上重启了一下 k3s 服务，这下完蛋了

集群的四个机器里一共有两个真 master，现在一台让我卸了 k3s，一台重启了服务。重启服务的一台服务永远起不来，因为连不上另一个 master。被我卸了的这台机器也无法重装 k3s，因为另一台 master 的服务没起来。这下彻底变成死锁了

集群里的另外两个 k3s-agent 现在也变成了不能动状态。一旦他们重启服务就再起不能了，于是集群彻底爆炸

用尽搜索大法，又和 gpt 高强度交互到半夜，依旧回天乏术。现在所有 master 的 etcd 全都是 dead 状态

折腾到了凌晨四点半，实在是撑不住了，明天还要考六级

集群爆炸的问题等明天有 k3s 大师再来修吧，我怕我再整出更大的爆炸，数据丢失了就真完蛋了

打印报告，交报告。然后去取我的 CD 😍，《Imaginary ENOZ featuring HARUHI》，首刷限定

拿到快递发现是一个非常大的盒子，以至于包里塞不下，也没法骑车手拿，只能弃车走回宿舍

我寻思我买的是 CD 啊，也不是唱片，怎么盒子这么大个

回宿舍之后激动开箱，发现原来是这样的：

这是一个非常大的封套，里面是一张卡纸内容，上面左下角嵌着 CD。泰酷辣！

之前没见过这种包装的，很喜欢，非常酷

晚上不想学习，开始研究博客。发现用 Hugo 作博客系统缺点非常多，它基本不开放什么 API，也不鼓励你自己写插件，顶多是写写主题

Hugo 虽然很轻，但是它更像是一种强调 “开箱即用” 的简单博客系统，不像 archlinux 一样为了高度定制化所以轻量级

因为用 Hugo 没法自定义 renderer 什么的，魔改空间不大，所以最后决定将博客搭建计划从基于 Hugo 转移到基于 Hexo

因为要用 Hexo 必须了解一点 Nodejs，要了解 Nodejs 必须了解一点 js，所以半夜在速通 js 语法。不求全记住，反正大概混个语法脸熟就行，遇到问题了问 AI，查文档就好了
